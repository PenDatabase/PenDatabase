## 🧠 About Me  

I am an engineer by discipline, not by title.

My work is centered on building systems that operate correctly under pressure—systems that serve, respond, and remain dependable when usage is no longer trivial.

I have worked across software and infrastructure, applying a consistent principle:  
every system must justify its design through performance, reliability, and clarity.

I have built and contributed to systems that:
- sustain **1,000+ concurrent users**  
- operate within **microservice architectures**  
- maintain stability through **structured error logging and monitoring**  

I approach backend engineering with a data-driven mindset.  
Every decision—schema, API, or architecture—is made with regard to:
- system reliability  
- latency  
- maintainability  

A system is not complete when it runs.  
It is complete when it can be relied upon.
